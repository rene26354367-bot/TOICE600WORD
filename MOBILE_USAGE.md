# 手機使用說明

## 結論

這個專案可以在手機上使用，但不建議直接用 `file://` 方式開啟。

正確做法是把專案放到一個可用 `http://` 或 `https://` 存取的靜態伺服器上。

## 為什麼不能直接用本機檔案

如果直接打開本機的 `index.html`，瀏覽器有機會因為 `file://` 載入限制，導致：

- `data.js` 沒有成功載入
- JavaScript 資料無法正確初始化
- 畫面看得到，但單字清單是空白

這不是介面問題，而是載入方式問題。

## 可行做法

### 1. 在電腦開本機伺服器，手機同 Wi-Fi 存取

在專案目錄執行：

```powershell
py -m http.server 8000
```

接著查詢電腦的區網 IP：

```powershell
ipconfig
```

找到 `IPv4 Address`，例如：

```text
192.168.1.20
```

然後在手機瀏覽器輸入：

```text
http://192.168.1.20:8000/index.html
```

### 2. 部署到靜態網站服務

可部署到以下任一平台：

- GitHub Pages
- Netlify
- Vercel
- Cloudflare Pages

部署後，手機直接用公開網址開啟即可。

### 3. 放到自己的網站空間

只要網站空間能提供靜態檔案，就可以上傳以下檔案：

- `index.html`
- `app.js`
- `data.js`
- `style.css`

## 建議

如果只是本機測試，建議用方法 1。

如果要長期在手機上使用，建議用方法 2，直接部署成公開靜態網站。

## 注意事項

- 手機與電腦若使用方法 1，必須連到同一個 Wi-Fi
- Windows 防火牆可能需要允許 Python 或對應連線埠
- 若頁面更新後手機看不到新內容，請重新整理或清除快取
